Professional Summary

JBT

At JBT I was part of the Automated Systems group that made Automated Guided Vehicles (AGVs). My 6+ years at JBT were spent on the vehicle software side. With my many years of C++ and embedded experience I quickly became the lead developer and "unofficial architect". I spent a lot of time refactoring code to make it more object-oriented and creating libraries for modularity and reuse. I added more and better ways to diagnose problems with the AGVs in the field to make the field engineers' job easier. All with the goal of reducing a large amount of accumulated technical debt.

I also wrote several internal tools for diagnostics and easier application development. These were both full C# applications using WPF/MVVM as well as command line tools

The vehicle software team was already using git when I arrived, but I was able to share my knowledge of "best practices" and also help other departments in adopting git

Pyramid Solutions

In my 17 years at Pyramid Solutions I was involved in many projects. I was part of the Data Communications department that focused on helping clients with software applications that involved network connectivity (PC applications, embedded devices, TCP/IP, Serial, etc). This exposed me to a lot of different development environments and IDEs. And because of this, I'm comfortable finding my way through any build environment or working with any IDE, even if I haven't used it before. I've also used several different source control schemes (although I prefer git).

The bulk of my involvement at Pyramid Solutions centered around the industrial protocol EtherNet/IP. I was responsible for Pyramid Solution's EtherNet/IP's protocol stack. It was sold as a standalone product, but I would also frequently do the integration into clients' embedded devices. This is where I learned what "code portability" really means and that not all TCP/IP stacks are created equal, i.e. BSD compatible.

I also put a lot of time towards ODVA, the standards organization that owns and promotes the EtherNet/IP protocol (CIP over Ethernet). I was involved with the conformance and interoperability of EtherNet/IP devices, meeting with EtherNet/IP device vendors to ensure compatibility between all devices. I was also instrumental in defining the specification for CIP-Modbus, a network translation layer when going from CIP to Modbus devices.

Wireshark

I was introduced to Wireshark during my time at Pyramid Solutions. At the time I was working on additional features to the EtherNet/IP protocol and the Wireshark dissection for it was incomplete. After getting over the hump of setting up the development environment (well documented, but lots of steps), I submitted my first patch. After a few rounds of review, the patch was accepted. I realized that EtherNet/IP is a "complicated" protocol from a dissection standpoint (involves both TCP and UDP and several dissection layers) and the best way to learn the Wireshark API to continue to improve the EtherNet/IP dissector was to try to understand and improve other protocols from existing bug reports. I eventually earned a spot as a core developer, with a lot of focus on refactoring the dissection engine for the transition to the Qt GUI interface and code reviewing submitted patches and trying to get them integrated into the main branch. I also attended a few Sharkfests (Wireshark developer/user conference)

Most of my work has already been submitted to the main branch (over 1900 commits!), but I keep a forked repository as that's how new functionality is currently added.
